# 本命絞り 前向き検証 運用メモ

## 明日からの流れ

1. 朝の荒れ監視TOP10を作る
2. 締切前にBOATERS AI・展示・オッズを取得する
3. 本命絞り条件が成立した時だけスマホ通知する
4. 通知された本命だけを前向き検証ログへ保存する
5. 結果が入った後、ログに的中・払戻・回収率を反映する

## 保存先

通知された本命は日別に保存する。

```text
data/output/forward_validation/core_focus_forward_YYYYMMDD.json
```

保存する主な項目:

- 日付
- レース場
- R
- 朝順位
- 締切時刻
- 展示後万舟率
- 本命絞りルールID
- 1着固定艇
- 元外頭候補2艇
- 軸候補
- 消し候補
- 買い目
- 点数
- 結果
- 払戻
- 的中
- 収支

## 結果反映

結果がデータベースへ入った後、次を実行すると前向き検証ログが更新される。

```bash
python3 scripts/update_core_focus_forward_results.py --date YYYY-MM-DD
```

全日分を更新する場合:

```bash
python3 scripts/update_core_focus_forward_results.py
```

## 判断基準

本命絞りの運用目標は回収率150%前後。
ただし少点数で的中率は低いため、最大連敗24程度は想定する。

前向き検証では、毎日次を見る。

- 買いR数
- 点数
- 的中
- 払戻
- 回収率
- 最大連敗
- 大当たり依存
